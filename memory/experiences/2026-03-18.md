2. **模型 API 找不到**: Gateway 重启后路由短暂丢失，报错 "No API key found for provider anthropic"
   - 原因: Gateway 重启时短暂丢失模型配置，去找了不存在的 Anthropic provider
   - 解决: 手动切换模型即可恢复
   - 配置文件里没有 anthropic 引用，是 OpenClaw 内部 fallback 机制

### 重要教训
- **Windows 上 openclaw gateway restart 不支持** — 因为 SIGUSR1 信号在 Windows 不存在
  - 需要手动 stop + start，或直接在 OpenClaw 应用里重启
- **Gateway 重启后配置可能被重置** — 修改 openclaw.json 后如果 Gateway 重启失败，可能回滚到旧配置
- **编辑 JSON 配置要小心** — ConvertTo-Json 在 PowerShell 里可能丢失嵌套结构，建议用增量编辑而不是整体重写
