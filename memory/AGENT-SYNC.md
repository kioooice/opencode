# 分身同步协议

**目的**：让所有 AutoClaw 分身共享记忆，切换分身不丢失上下文。

## 核心原则

1. **记忆存储在文件里，不是脑子里的"心理笔记"**
2. **每个分身启动时必须读取共享记忆**
3. **用户主动控制同步时机**（不是自动同步）

## 同步流程

```
┌─────────────────────────────────────────────┐
│  正常使用                                    │
│  - 不需要频繁同步                            │
│  - 重要信息写入 memory/ 目录                 │
└─────────────────────────────────────────────┘
                    ↓
         用户说"同步"或"保存状态"
                    ↓
┌─────────────────────────────────────────────┐
│  关闭前同步                                  │
│  git add .                                   │
│  git commit -m "sync: session end"           │
│  git push                                    │
└─────────────────────────────────────────────┘
                    ↓
              换电脑/重启网关
                    ↓
┌─────────────────────────────────────────────┐
│  启动后同步                                  │
│  git pull                                    │
│  读取 MEMORY.md + world/facts + insights    │
└─────────────────────────────────────────────┘
```

## 强制启动流程

每个分身**必须**在第一次响应前执行：

```
1. 读取 MEMORY.md（长期记忆索引）
2. 读取 memory/world/facts.md（用户偏好和项目知识）
3. 读取 memory/insights/insights.md（学到的规律）
4. 读取最近 3 天的 memory/YYYY-MM-DD.md（近期经历）
```

## 写入规则

| 信息类型 | 写入位置 | 时机 |
|----------|----------|------|
| 用户偏好、项目知识 | `memory/world/facts.md` | 识别到时 |
| 完成的任务、学到的东西 | `memory/YYYY-MM-DD.md` | 用户请求时 |
| 提炼的规律、最佳实践 | `memory/insights/insights.md` | 反思时 |

## 同步命令

### 用户说"同步"或"保存状态"
```powershell
git add .
git commit -m "sync: session end"
git push
```

### 用户说"拉取"或"同步记忆"
```powershell
git pull
```

## 验证检查

如果你不确定分身是否加载了记忆，问：

> "我的 GitHub 地址是什么？最近在做什么项目？"

正确回答 = 记忆加载成功
无法回答 = 需要检查记忆系统
